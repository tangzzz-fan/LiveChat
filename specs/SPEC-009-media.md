# SPEC-009 — 媒体消息：大文件不走消息管道

> 状态: Draft | Milestone: M3 | 依赖: SPEC-003, 004 | 被依赖: 011(附件加密)

## 1. 背景与动机（Why）

一张照片 3MB = 3 万条文本消息的体积。让 blob 走 WebSocket 消息管道会同时
毒害两边：网关内存被大帧占满（背压全乱）、消息延迟被传输拖垮。
**第一原则：控制面（消息）与数据面（文件字节）彻底分离。** 所有大厂 IM 无一例外。

## 2. 核心挑战与典型解法

### 挑战 A：上传-发送的解耦时序

**解法：先传后发（upload-then-send，WhatsApp/Telegram/iMessage 通用）：**

```
① 客户端压缩/转码 → ② API 换取预签名 PUT URL → ③ 直传 MinIO(S3 协议)
→ ④ 通过普通消息管道发 ImageContent{object_key, width, height, size,
     sha256, blurhash} —— 消息里只有元数据引用, 复用 SPEC-003 全部可靠性
→ ⑤ 接收方拿 object_key 换预签名 GET URL 下载
```

- 服务端（网关/msgsvc）**永远不碰文件字节**——预签名 URL 让客户端与对象
  存储直连，这就是"数据面旁路"；
- 秒传/去重：按 `sha256` 查已有对象，命中则跳过上传直接发引用
  （微信"秒发"原图的原理）；
- 引用计数与生命周期：消息删除/过期不立即删对象，异步 GC 扫描无引用对象
  （先记账后清扫，避免分布式删除竞态）。

### 挑战 B：移动网络上传的可靠性

- 3MB 文件在地铁里一次 PUT 的成功率感人。**分片上传**（S3 multipart，
  5MB 以下单片即可，视频必分片）+ 断点续传（记录已完成分片，重试只补缺）；
- 上传任务挂到 SPEC-004 的 outbox 体系：消息状态机扩展为
  `pending_upload → uploading(带进度) → pending → sent...`，
  杀 App 重开后续传（URLSession background configuration 正好学一遍）；
- 失败降级：上传重试预算耗尽 → 消息转 failed，用户可点重试（复用原分片）。

### 挑战 C：图片体验工程

- 客户端发送前处理：长边 ≤ 2048 压缩（"原图"选项另传原件）、HEIC→JPEG
  兼容转码、EXIF 剥离（**GPS 信息必须剥，隐私红线**）；
- 占位体验：`blurhash`（默认 4×3 分量、base83 编码后约 30 字符 ≈ 30 字节
  的模糊缩略图编码）随消息元数据走，气泡瞬间显示模糊图 → 下载完成淡入
  清晰图，消灭白块闪烁；
- 缩略图策略：服务端不做转码集群（超出学习范围），客户端上传时顺带传一张
  512px 缩略图对象，列表/气泡先用缩略图，点开才拉原图——省 90% 流量；
- 下载缓存：GRDB 存元数据，文件落 `Caches/media/{sha256}`，LRU 上限 500MB。

### 挑战 D：语音消息

录音（AVAudioEngine, Opus/AAC 64kbps）→ 与图片同管道；元数据带时长与
波形采样数组（气泡直接画波形，不用下载后解析）；播放进度全局单实例
（同时只播一条）。

## 3. 范围

**In**：MinIO 部署进 docker-compose、预签名 URL API（含权限校验：只有会话
成员能取 GET URL）、分片/续传、图片全链路（压缩/EXIF/blurhash/缩略图/缓存）、
语音消息、GC 任务。
**Out**：视频转码与流式播放（backlog，管道已兼容）、CDN（云部署被 D8 排除，
文档一句话说明生产上 GET URL 换 CDN 签名 URL 即可）、文档/任意文件类型（backlog）。

## 4. 验收标准

1. 网关内存实验：连发 50 张图期间，网关每连接内存占用与纯文本场景无差异
   （数据面旁路的直接证明，指标对比）。
2. 断点续传实验：上传 20MB 视频文件至 50% 时杀 App → 重开 → 从 ~50% 续传
   成功（mock 慢速网络 + 分片日志为证）。
3. 秒传：同一张图第二次发送，零字节上传（MinIO 访问日志证明），消息正常到达。
4. 隐私：上传后的对象用 exiftool 检查，GPS 字段不存在。
5. 权限：非会话成员用泄露的 object_key 请求 GET URL → 403（API 测试断言）。
6. 弱网体验：Network Link Conditioner 3G 档，发图后气泡立即出现 blurhash
   占位与进度环，无 UI 冻结。

## 5. 测试计划

分片状态机、GC 引用计数单测；MinIO testcontainer 集成测试；验收 1/2/6 为
真机 runbook；权限矩阵进 API 自动化测试。
